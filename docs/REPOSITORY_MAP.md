## 📍 File Locations by Purpose

### Need to...

#### Read Documentation?
- **Overview** → `README.md`
- **Specifications** → `docs/specifications/`
- **Guides** → `docs/guides/`
- **API Reference** → `docs/api/`
- **Examples** → `docs/examples/` or `examples/`

#### Write Code?
- **Core library** → `packages/core/src/`
- **Database adapter** → `packages/adapters/{orm}/src/`
- **Extension** → `packages/extensions/{name}/src/`
- **Infrastructure** → `packages/infrastructure/{name}/src/`

#### Write Tests?
- **Unit tests** → Same directory as code (`*.test.ts`)
- **Integration tests** → `tests/integration/`
- **E2E tests** → `tests/e2e/`
- **Test data** → `tests/fixtures/`

#### Configure Something?
- **Root package** → `package.json`
- **Monorepo** → `lerna.json`
- **TypeScript** → `tsconfig.json`
- **Linting** → `.eslintrc.json`
- **Formatting** → `.prettierrc.json`
- **CI/CD** → `.github/workflows/`

#### Learn How to Use?
- **Quick start** → `README.md`
- **Detailed guide** → `docs/guides/GETTING_STARTED.md`
- **Full example** → `examples/{use-case}/`
- **API docs** → `docs/api/`

## 🎯 Development Workflow

### Adding a New Feature
```
1. Read CONTRIBUTING.md
2. Create feature branch
3. Write code in packages/{package}/src/
4. Write tests in packages/{package}/tests/
5. Update docs in docs/
6. Create PR
```

### Adding a New Package
```
1. Create directory in packages/{category}/{name}/
2. Copy package.json template
3. Write code in src/
4. Write tests in tests/
5. Update root package.json workspaces
6. Run lerna bootstrap
```

### Writing Documentation
```
1. Specifications → docs/specifications/
2. User guides → docs/guides/
3. API docs → TSDoc in source code (auto-generated)
4. Examples → docs/examples/ or examples/
```
